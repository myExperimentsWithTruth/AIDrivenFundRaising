# Audit report — template

Generated by the audit agent following SOP §9.2. Filename: `archive/audit_<YYYYMMDD_HHMMSS>.md`. The audit agent never edits anything; it only reads and reports.

Two possible outcomes: APPROVED or DISCREPANCY.

---

## DISCREPANCY template

```
AUDIT REPORT — DISCREPANCY

Timestamp: <ISO 8601>
Audited file: daily_updates/YYYYMMDD_update.md
Excel snapshot: Fundraise_Reconciliation_Sheet.xlsx (modified <timestamp>)

FAILED checks: <count>

Check 1: <name of check>
  Expected: <what the audit agent re-derived>
  Observed: <what the message / Excel actually shows>
  Offending row(s): <pointer into Excel or message>
  Likely cause: <brief>

Check 2: <name of check>
  Expected: ...
  Observed: ...
  Offending row(s): ...
  Likely cause: ...

Resolution required:
  - Primary agent investigates the rows listed above.
  - Apply minimum-necessary fix per SOP §9.4. Take Excel backup (§8.5) first.
  - Re-run §8.1 verification and then re-run audit (§9.2).
  - Do not send the daily message until audit reports APPROVED.

The audit agent has not edited anything.
```

---

## Notes

- The audit agent re-loads `RECON_SOP.md` at the start of every pass. It does not assume any prior knowledge of today's work.
- The audit agent re-reads the bank-stmt PDFs from disk; it does not trust the bank-sheet rows the primary agent imported.
- Random-sample re-import: five rows are picked at random from the most recent bank statement. Any mismatch is a discrepancy.
- Resolution flow is SOP §9.4: investigate → minimum fix → re-audit → APPROVED → day closes.
